Quirks

1. Name shadowing and simplifyfull

When defining a Sagemath function, it is important to avoid name shadowing. For example 

x = function("x")

uses a variable x and a function name x at the same time. I found it hard to catch bugs due to name shadowing, so to avoid this, I decided to precede the internal name of a function with an underscore, but keep the LaTeX name the same as the original name. The next implementation achieves this behavoir. 

def function(name):
    return sage.symbolic.function_factory.function("_" + name, latex_name=name)

This is another implementation without the underscore in the name. 

def function(name):
    return sage.symbolic.function_factory.function(name, latex_name=name)

This results in a bug when using a Greek letter as a name. 

var("t")

x = sage.symbolic.function_factory.function("Y", latex_name="\\xi")(t)
print(latex(x))
res = x.simplify()
print(latex(res))

x = sage.symbolic.function_factory.function("\\phi", latex_name="\\xi")(t)
print(latex(x))
res = x.simplify()
print(latex(res))

x = sage.symbolic.function_factory.function(r"\phi", latex_name="\\xi")(t)
print(latex(x))
res = x.simplify()
print(latex(res))

t
\xi\left(t\right)
\xi\left(t\right)
\xi\left(t\right)
\phi\left(t\right)
\xi\left(t\right)
\phi\left(t\right)

So for this reason I remove the underscore in function.
